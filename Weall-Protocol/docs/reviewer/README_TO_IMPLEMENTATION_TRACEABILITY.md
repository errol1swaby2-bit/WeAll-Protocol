# README to Implementation Traceability

## Purpose

This document maps major README claims to repository evidence and explicitly distinguishes implementation, local evidence, generated artifacts, proof templates, disabled surfaces, and open external blockers. It is a reviewer navigation document, not a readiness escalation.

## Current claim boundary

Current allowed claim: **WeAll is a pre-public-testnet protocol implementation under active hardening, with local/devnet/public-observer-oriented evidence present and public beta readiness still blocked by explicit external observer, replay, validator/operator, storage, legal, upgrade-execution, and helper-topology gates.**

- `public_beta_ready` is `false`.
- Public mainnet readiness is not claimed.
- Public multi-validator BFT readiness is not claimed.
- Public validator safety is not claimed.
- Live economics readiness is not claimed.
- Automatic upgrade readiness is not claimed.
- Production helper execution readiness is not claimed.
- Legal/compliance approval is not claimed.
- Public storage-market readiness is not claimed.

## Traceability status legend

| Status | Meaning | Reviewer boundary |
|---|---|---|
| Implemented + tested | Source exists and targeted tests or frontend source checks cover the claim. | Supports bounded repository review only. |
| Implemented + local evidence | Source and local generated/docs evidence exist, but external transcript evidence is still absent. | Does not close external blockers. |
| Generated evidence | A checked-in generator emits deterministic evidence. | Proves repository consistency only. |
| Documentation/runbook | A reviewer/operator path is documented. | Needs runtime transcript before stronger claims. |
| Template only | A schema/runbook exists for future evidence capture. | Not completed external evidence. |
| Record only | The protocol records metadata; execution is not enabled. | Do not infer automatic apply, migration, or rollback. |
| Disabled by launch gate | The surface is deliberately locked or not launch-enabled. | Do not describe it as available. |
| External evidence required | A claim cannot be strengthened without independent transcript/attestation. | Open blocker remains visible. |
| Future hardening | A plan/proof slot exists for later mainnet-readiness work. | Not in scope for bounded rehearsal. |
| Unsupported/needs correction | No support was found or wording was too broad. | README must be narrowed before review. |
