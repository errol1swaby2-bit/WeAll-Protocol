# M-1 through M-10 mechanics implementation batch

This batch shifts the v1.5 workstream from reviewer packaging to executable mechanics.
It does **not** enable public validators, live economics, automatic software upgrades,
or production helper execution.

## Truth boundaries

- Public validators remain disabled.
- Public multi-validator BFT readiness is not claimed.
- Live economics remains locked.
- Balance transfers, tips, rewards, and treasury spend remain locked.
- Protocol upgrades remain record-only.
- Helper/parallel execution remains unclaimed for production.
- Automatic software upgrade apply/migration/rollback is not implemented.
